# Isabelle Auto-Repair Pipeline

Automatically repairs [Archive of Formal Proofs (AFP)](https://www.isa-afp.org/) theories that fail to build due to version incompatibilities between Isabelle releases (e.g. AFP 2023 → AFP 2024).

Three pipelines are provided:
1. **LLM repair** — uses an LLM (GPT) to generate source-level fixes
2. **Sledgehammer repair** — uses Mirabelle/Sledgehammer to find new proof tactics
3. **Context-aware LLM repair** — re-runs LLM repair for persistent failures with dependency-change context

---

## Requirements

### Python dependencies

```bash
pip install openai openpyxl
```

### Configure `setup.py`

Edit `setup.py` at the repo root to match your local environment:

| Variable | Description |
|---|---|
| `API_KEY` | OpenAI API key |
| `SOURCE_AFP` | AFP source theories  (e.g. afp2023/thys)|
| `TARGET_AFP` | AFP target theories (e.g. afp2024/thys) |
| `SOURCE_ISABELLE` | Path to the source Isabelle binary (e.g. Isabelle2023/bin/Isabelle) |
| `TARGET_ISABELLE` | Path to the target Isabelle binary |
| `OUTPUT_ROOT` | Root of the labeled failure dataset (see below) |
| `MIRABELLE_OUTPUT` | Directory where Mirabelle writes its log files |
| `XLSX_PATH` | Output Excel log file path |
| `DEPENDENCY_OUTPUT` | Where `collect_dep.py` writes dependency fact files |
| `CHANGES_OUTPUT` | Where `collect_dep_changes.py` writes dependency-change files |

### Error dataset — `OUTPUT_ROOT`

All pipelines expect pre-extracted build failures under `OUTPUT_ROOT`:

```
OUTPUT_ROOT/
  <SessionName>/
    <SessionName>@<TheoryName>.json
    <SessionName>@<SubSession>@<Theory>.json   # nested session
```

Each JSON file contains a `failures` array with error positions and messages:

```json
{
  "failures": [
    { "pos": 339, "msg": "Undefined fact \"wfPUNIVI\"\nAt command \"apply\" (line 339)" },
    { "pos": 345, "msg": "Failed to finish proof\nAt command \"by\" (line 345)" }
  ]
}
```

## Pipeline 1 — LLM Repair

Reads each error, sends the erroneous source snippet to an LLM (GPT), applies block-level replacement fixes, and verifies by rebuilding with Isabelle. Retries up to `max_attempts` times.

```bash
./llm_repair.sh                              # repair all sessions in OUTPUT_ROOT
./llm_repair.sh --session ADS_Functor        # repair a single session
./llm_repair.sh --session CakeML/Tests       # nested session
./llm_repair.sh --additional_info "wfPUNIVI was renamed to wfP_induct"
./llm_repair.sh --show-configured-list       # preview queued sessions
```

**What it can fix:** any error where a correct replacement can be generated by the LLM — renamed lemmas, changed syntax, refactored API, etc.

Results are written to the **`llm-repair`** sheet of `XLSX_PATH`.

---

## Pipeline 2 — Sledgehammer Repair

Targets proof step failures specifically. For each failing proof command, it inserts a `sorry` placeholder, runs [Mirabelle](https://isabelle.in.tum.de/doc/jedit.pdf) with `sledgehammer` to search for a working proof, and records any found proof.

```bash
./sledgehammer_repair.sh                     # repair all sessions in OUTPUT_ROOT
./sledgehammer_repair.sh --session ADS_Functor
./sledgehammer_repair.sh --show-configured-list
```

**What it can fix:**

| Error type | Example | Handled |
|---|---|---|
| `by` tactic failure | `by (simp add: foo)` | Yes — comments out tactic, inserts sorry, runs sledgehammer |
| `apply` tactic failure | `apply (auto simp: bar)` | Yes — same as above |
| Structured `proof`/`qed` block | `proof -\n  ...\nqed` | Yes — wraps entire block in sorry, runs sledgehammer |

**What it cannot fix:**

- Errors not caused by a failing proof step (e.g. type errors, missing imports, renamed constants) — these are `CannotBeFixed` and skipped
- Cases where Sledgehammer times out or cannot find a proof within the configured timeout

Results are written to the **`sledgehammer`** sheet of `XLSX_PATH`.

---

## Pipeline 3 — Context-Aware LLM Repair

A follow-up pipeline for theories that still fail after both Pipeline 1 and Pipeline 2. It enriches the LLM prompt with structured information about how the Isabelle/HOL library lemmas that the proof depends on have changed between versions.

### Step 1 — Collect dependency facts

For each failing theory, run `collect_dep.py` to record which library lemmas each failing proof step depends on (requires an Isabelle server):

```bash
python context-repair/collect_dep.py --session Actuarial_Mathematics
```

Output: `DEPENDENCY_OUTPUT/Isabelle2023-To-Isabelle2024/<Session>/<Theory>_dep.txt`

### Step 2 — Collect dependency changes

Diff the library lemmas recorded in Step 1 against the source/target Isabelle releases to produce a structured change summary:

```bash
python context-repair/collect_dep_changes.py
```

Output: `CHANGES_OUTPUT/Isabelle2023-To-Isabelle2024/<Session>/<Theory>_dep.txt`

Each change file groups diffs by error block `[N] line <L>:`, matching the format of the LLM error prompt, so the model knows which changed lemma is relevant to which error.

### Step 3 — Run context-aware repair

```bash
python context-repair/context_repair_driver.py [--max-attempts N] [--dry-run]
```

- Reads all theories that eventually fail in both `llm-repair` and `sledgehammer` sheets
- For each, looks for a corresponding changes file from Step 2
- If found, passes the change diffs as `additional_info` in the LLM prompt and retries repair

Results are written to the **`context-repair`** sheet of `XLSX_PATH`.

## Output

All pipelines log to the same Excel file (`XLSX_PATH`), each in its own sheet:

- **`llm-repair`** sheet — Pipeline 1 results
- **`sledgehammer`** sheet — Pipeline 2 results
- **`context-repair`** sheet — Pipeline 3 results

Each row records one repair attempt:

| Column | Description |
|---|---|
| `session+theory` | e.g. `CakeML/Tests/Compiler_Test.thy` |
| `attempt` | Attempt number |
| `error message` | The error extracted from the build failure |
| `fixes` | Fix(es) found (line number + replacement) |
| `fail/success` | Outcome |
| `elapsed_seconds` | Wall-clock time |

## Project structure

```
auto-repair-pipeline/
├── setup.py                         # All configurable paths and API key
├── llm_repair.sh                    # Entry point for Pipeline 1
├── sledgehammer_repair.sh           # Entry point for Pipeline 2
├── llm-repair/
│   ├── repair_driver.py             # Driver: iterates sessions and theories
│   ├── llm_repairer_multiline.py    # Core repair loop (LLM, apply, build, retry)
│   ├── failure_extractor.py         # Reads JSON errors and extracts source snippets
│   ├── build_error_message_extractor.py  # Filters build output to target-theory errors
│   ├── session_lister.py            # Enumerates sessions from OUTPUT_ROOT
│   └── excel_creater.py             # Appends rows to the Excel log
├── sledgehammer-repair/
│   ├── driver.py                    # Driver: iterates sessions and theories
│   ├── classifier.py                # Classifies error type and dispatches to handler
│   ├── handler.py                   # Modifies theory file (inserts sorry placeholder)
│   ├── mirabelle_runner.py          # Runs Isabelle Mirabelle with sledgehammer action
│   ├── parser.py                    # Parses error messages and Mirabelle logs
│   ├── failure_extractor.py         # Reads JSON errors for sledgehammer pipeline
│   └── attempt_writer.py            # Writes results to Excel
└── context-repair/
    ├── context_repair_driver.py     # Entry point for Pipeline 3
    ├── collect_dep.py               # Collects dependency facts via Isabelle server
    ├── collect_dep_changes.py       # Diffs dependency lemmas across Isabelle versions
    ├── static_proof_check.py        # Static analysis to identify proof-state commands
    ├── fact_collector.py            # Queries Isabelle for facts used in a proof step
    └── parse_thy_to_json.py         # Parses .thy files into structured JSON
```
